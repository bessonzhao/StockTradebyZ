# 选股逻辑详解

## 1. 选股框架设计

AgentTrader 采用了模块化、向量化、Numba 加速的选股框架，设计原则如下：

### 1.1 核心设计原则

- **每个指标/条件独立成 Filter dataclass**
  - `__call__(hist) -> bool`：逐行点查（回调/调试用）
  - `vec_mask(df) -> ndarray[bool]`：全量向量化（批量回测用）

- **PipelineSelector 基类**
  - `prepare_df(df)`：预计算所有中间列，返回含 `_vec_pick` 的 df
  - `passes_df_on_date(df, date)`：单日判断
  - `vec_picks_from_prepared(df)`：从预计算列批量获取通过日期

- **Numba 加速**
  - KDJ 递推、砖型图核心循环、连续绿柱计数均使用 `@njit` 加速

### 1.2 关键技术点

- **向量化计算**：使用 NumPy 数组进行批量计算，提高效率
- **Numba 加速**：将核心计算函数编译为机器码，加速数值计算
- **模块化设计**：便于添加新的选股策略和指标
- **无未来函数**：所有计算均基于历史数据，避免未来信息泄漏
- **并行化潜力**：设计支持多线程/多进程并行计算

## 2. 核心选股策略

AgentTrader 实现了两种主要选股策略：B1 选股器和砖型图选股器。

### 2.1 B1 选股器

B1 选股器基于 KDJ 分位、知行线和周线多头排列等指标，适合趋势性选股。

#### 2.1.1 核心过滤器

| 过滤器 | 作用 | 参数 |
|--------|------|------|
| KDJQuantileFilter | J 值低位过滤 | j_threshold: J 值绝对阈值<br>j_q_threshold: J 值历史分位阈值<br>kdj_n: KDJ 计算周期 |
| ZXConditionFilter | 知行线条件过滤 | zx_m1-zx_m4: 四条均线参数<br>zxdq_span: 知行线计算周期<br>require_close_gt_long: 收盘价>长期均线<br>require_short_gt_long: 快线>长期均线 |
| WeeklyMABullFilter | 周线多头排列 | wma_short: 短期周均线<br>wma_mid: 中期周均线<br>wma_long: 长期周均线 |
| MaxVolNotBearishFilter | 最大成交量非阴线 | n: 考察周期 |

#### 2.1.2 选股逻辑

1. **KDJ 分位过滤**：今日 J 值 < 阈值或处于历史低分位
2. **知行线条件**：收盘价 > 长期均线且快线 > 长期均线
3. **周线多头排列**：短期周均线 > 中期周均线 > 长期周均线
4. **最大成交量非阴线**：过去 n 个交易日内成交量最大的一天不是阴线

### 2.2 砖型图选股器

砖型图选股器基于砖型图形态、知行线和周线多头排列等指标，适合捕捉反转信号。

#### 2.2.1 核心过滤器

| 过滤器 | 作用 | 参数 |
|--------|------|------|
| BrickPatternFilter | 砖型图形态过滤 | daily_return_threshold: 单日涨幅阈值<br>brick_growth_ratio: 砖型图增长比例<br>min_prior_green_bars: 连续绿柱数<br>brick_params: 砖型图计算参数 |
| ZXDQRatioFilter | 知行线比例过滤 | zxdq_ratio: 收盘价与知行线比值<br>zxdq_span: 知行线计算周期 |
| ZXConditionFilter | 知行线条件过滤 | 同上 |
| WeeklyMABullFilter | 周线多头排列 | 同上 |

#### 2.2.2 选股逻辑

1. **砖型图形态**：
   - 今日涨幅 < 阈值
   - 今日红柱（brick > 0）
   - 昨日绿柱（brick[-1] < 0）
   - 今日红柱高度 ≥ 昨日绿柱绝对高度 × 增长比例
   - 红柱前连续绿柱数 ≥ 最小连续绿柱数

2. **知行线比例**：收盘价 < 知行线 × 比值
3. **知行线条件**：快线 > 长期均线
4. **周线多头排列**：短期周均线 > 中期周均线 > 长期周均线

## 3. 选股流程

### 3.1 数据准备阶段

1. **加载配置文件**：读取 `rules_preselect.yaml` 中的选股参数
2. **读取原始数据**：从 `data/raw` 目录读取股票日线数据
3. **数据预处理**：
   - 统一列名和日期格式
   - 排序并截断数据
   - 计算换手率等辅助指标

### 3.2 流动性池构建

1. **计算换手率**：计算每只股票的日均换手率
2. **构建流动性池**：选取换手率前 `top_m` 只股票作为流动性池

### 3.3 选股执行阶段

1. **确定选股日期**：
   - 未指定日期：使用最晚可用交易日
   - 指定日期：向前搜索最近的可用交易日

2. **运行 B1 选股器**：
   - 对流动性池中的每只股票应用 B1 选股规则
   - 生成 B1 策略候选股票列表

3. **运行砖型图选股器**：
   - 对流动性池中的每只股票应用砖型图选股规则
   - 生成砖型图策略候选股票列表
   - 按砖型图增长倍数排序

4. **去重处理**：
   - 合并两个策略的候选列表
   - 去重，保留首次命中的策略

### 3.4 结果输出

1. **生成候选文件**：将候选股票列表保存到 `data/candidates/candidates_latest.json`
2. **候选股票信息**：包含股票代码、选股日期、策略类型、收盘价、换手率等信息

### 4.1 KDJ 指标

```python
def compute_kdj(df: pd.DataFrame, n: int = 9) -> pd.DataFrame:
    """计算 KDJ 指标"""
    low_n  = df["low"].rolling(window=n, min_periods=1).min()
    high_n = df["high"].rolling(window=n, min_periods=1).max()
    rsv    = ((df["close"] - low_n) / (high_n - low_n + 1e-9) * 100).to_numpy()
    # 使用 Numba 加速 KDJ 递推计算
    K, D, J = _kdj_core(rsv)
    return df.assign(K=K, D=D, J=J)
```

### 4.2 知行线

```python
def compute_zx_lines(df: pd.DataFrame, m1: int, m2: int, m3: int, m4: int, zxdq_span: int) -> tuple:
    """计算知行线指标"""
    close = df["close"].astype(float)
    # 双指数平滑移动平均线
    zxdq = close.ewm(span=zxdq_span, adjust=False).mean().ewm(span=zxdq_span, adjust=False).mean()
    # 四均线平均值
    zxdkx = (close.rolling(m1).mean() + close.rolling(m2).mean() + 
             close.rolling(m3).mean() + close.rolling(m4).mean()) / 4.0
    return zxdq, zxdkx
```

### 4.3 周线收盘价

```python
def compute_weekly_close(df: pd.DataFrame) -> pd.Series:
    """日线转周线收盘价"""
    close = df.set_index("date")["close"].astype(float)
    # 按 ISO 年+周分组，取每组最后一个交易日的收盘价
    year_week = close.index.isocalendar().year.astype(str) + "-" + close.index.isocalendar().week.astype(str).str.zfill(2)
    weekly = close.groupby(year_week).last()
    # 将 index 换回真实日期
    last_date_per_week = close.groupby(year_week).apply(lambda s: s.index[-1])
    weekly.index = pd.DatetimeIndex(last_date_per_week.values)
    return weekly.dropna()
```

### 4.4 周线均线多头排列

```python
def compute_weekly_ma_bull(df: pd.DataFrame, ma_periods: tuple) -> pd.Series:
    """计算周线均线多头排列"""
    weekly_close = compute_weekly_close(df)
    s, m, l = ma_periods
    ma_s = weekly_close.rolling(s, min_periods=s).mean()
    ma_m = weekly_close.rolling(m, min_periods=m).mean()
    ma_l = weekly_close.rolling(l, min_periods=l).mean()
    bull = (ma_s > ma_m) & (ma_m > ma_l)
    # 转换为日线频率
    daily_index = pd.DatetimeIndex(df["date"])
    bull_daily = bull.astype(float).reindex(daily_index).ffill().fillna(0.0).astype(bool)
    return bull_daily
```

### 4.5 砖型图

```python
def compute_brick_chart(df: pd.DataFrame, n: int, m1: int, m2: int, m3: int, 
                        t: float, shift1: float, shift2: float, 
                        sma_w1: int, sma_w2: int, sma_w3: int) -> pd.Series:
    """计算砖型图指标"""
    # 使用 Numba 加速砖型图计算
    arr = _compute_brick_numba(
        df["high"].to_numpy(), df["low"].to_numpy(), df["close"].to_numpy(),
        n, m1, m2, m3, float(t), float(shift1), float(shift2),
        sma_w1, sma_w2, sma_w3,
    )
    return pd.Series(arr, index=df.index, name="brick")
```

## 7. 扩展与自定义

### 7.1 添加新的选股策略

1. 在 `Selector.py` 中继承 `PipelineSelector` 类
2. 实现 `prepare_df` 方法，预计算所有中间列
3. 在 `select_stock.py` 中添加新策略的运行函数
4. 在配置文件中添加新策略的配置项

### 7.2 添加新的过滤器

1. 创建新的 Filter dataclass，实现 `__call__` 和 `vec_mask` 方法
2. 在选股器中使用新的过滤器
3. 根据需要添加配置参数

### 7.3 调整选股参数

通过修改 `config/rules_preselect.yaml` 文件，可以调整各种选股参数，如：

- 调整流动性池大小
- 修改 KDJ 阈值
- 调整知行线参数
- 启用/禁用特定策略

## 8. 性能优化

### 8.1 向量化计算

- 使用 NumPy 数组替代 Pandas DataFrame 进行核心计算
- 避免循环，使用向量化操作
- 使用 Numba 加速数值计算

### 8.2 预计算优化

- 一次性预计算所有中间列
- 避免重复计算相同指标
- 利用缓存机制提高回测效率

### 8.3 并行化设计

- 支持多线程/多进程并行计算
- 选股过程可分布式执行
- 适合大规模回测和实时选股

## 9. 总结

AgentTrader 采用了先进的量化选股框架，结合了 KDJ 分位、知行线、周线多头排列等多种指标，实现了高效、准确的选股功能。框架设计模块化、向量化、Numba 加速，具有良好的扩展性和性能表现。

通过调整配置参数，可以灵活适应不同的市场环境和投资风格。同时，框架支持添加新的选股策略和指标，便于持续优化和改进选股逻辑。
